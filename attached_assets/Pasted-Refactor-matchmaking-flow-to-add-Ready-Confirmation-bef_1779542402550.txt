Refactor matchmaking flow to add Ready Confirmation before revealing cut links.

Current issue:
Too many users get matched but do not press Done Cut / do not send proof.
We need a fair pre-confirmation step before showing partner links.

━━━━━━━━━━━━━━━━━━
NEW MATCH FLOW
━━━━━━━━━━━━━━━━━━

When 2 users are matched:

DO NOT reveal partner cut links immediately.

Instead, send both users a confirmation message:

🎯 Cut buddy found!

Before we show the cut link, please confirm you are ready.

Buttons:
✅ Ready To Cut
❌ Cancel Match

━━━━━━━━━━━━━━━━━━
READY LOGIC
━━━━━━━━━━━━━━━━━━

1. If BOTH users press ✅ Ready To Cut:
- mark both users as ready
- reveal both partner cut links
- start normal active match flow
- start Done Cut / proof timer
- continue existing proof flow:
  • Done Cut
  • choose TikTok account
  • upload screenshot proof
  • partner approve/reject

2. If ONE user presses ❌ Cancel Match before both are ready:
- cancel the match
- do NOT punish anyone
- do NOT apply cooldown
- do NOT count as ghost/incomplete strike
- clear match state for cancelling user
- clear match state for ready user

If the other user had already pressed Ready:
- auto return that ready user back into matchmaking queue
- try to find a new partner for them
- BUT do NOT auto-ready them
- they must press ✅ Ready To Cut again for the new match

3. If neither user presses Ready within timeout:
- cancel match
- no punishment
- both users leave match/queue
- user must submit link again if they want to match again

━━━━━━━━━━━━━━━━━━
IMPORTANT AUTO-MATCH RULE
━━━━━━━━━━━━━━━━━━

Auto rematch is ONLY allowed when:
- user already pressed ✅ Ready To Cut
- but partner cancelled before link reveal

Do NOT auto rematch after:
- successful completed swap
- proof approved
- proof rejected
- proof timeout
- match timeout after link reveal
- inactivity/ghost punishment
- stuck cleanup

After completed swap:
- stop everything
- clear active match
- user must manually submit a new cut link to join queue again

━━━━━━━━━━━━━━━━━━
PUNISHMENT RULE
━━━━━━━━━━━━━━━━━━

No punishment before links are revealed.

Punishment only applies AFTER both users pressed Ready and links were revealed.

Punish users if:
- did not press Done Cut after Ready
- did not choose TikTok account
- did not upload proof
- did not approve/reject proof
- ghosted after link reveal

Keep existing punishment system:
- 24h cooldown where applicable
- 3 repeated serious offences = permanent ban
- admin notifications continue
